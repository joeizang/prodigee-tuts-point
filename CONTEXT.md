# Prodigee Tuts Point

Prodigee Tuts Point is a private learning system for building project-ready mastery across server-side languages, frameworks, algorithms, and engineering practice.

## Language

**Project-Backed Mastery Loop**:
The primary learning loop where project milestones determine what theory, examples, and exercises the learner studies next.
_Avoid_: Passive tutorial browsing, reference-first learning

**Exercise-First Pressure**:
Frequent coding tasks used to force active recall and skill formation inside the broader project-backed path.
_Avoid_: Challenge-only learning, isolated puzzles

**Theory for Mastery**:
Clearly sequenced explanations, mental models, tradeoffs, and production context required before and during coding practice.
_Avoid_: Shallow intros, trivia, syntax-only notes

**First Slice Language**:
C# is the first implementation language used to prove the learning loop before expanding to TypeScript and Swift.
_Avoid_: Starting all languages before proving the curriculum loop

**Soft Lock**:
A progression rule that recommends prerequisites and warns about gaps without preventing the learner from continuing.
_Avoid_: Hard gate, forced sequence

**Evidence-Based Mastery**:
A progress model that infers competence from exercise attempts, project use, review recall, failures, and time decay.
_Avoid_: Page completion, fake precision

**Serious IDE/Book Hybrid**:
The product experience that combines technical-book depth with an IDE-like coding workspace.
_Avoid_: Course marketplace, childish game skin

**Restrained Gamification**:
Progress feedback that reinforces mastery through streaks, milestones, levels, and review pressure without replacing the serious learning tone.
_Avoid_: Badge clutter, points as the goal

**Local Learner Profile**:
A local identity used to attach progress, attempts, preferences, and mastery state without requiring authentication.
_Avoid_: User account, login identity

**Personal Study Note**:
A learner-authored note attached to a lesson, exercise, project, milestone, concept, or source reference.
_Avoid_: Published curriculum content

**Source Reference**:
A metadata pointer from curriculum to a book, chapter, page range, or topic that supports deeper study.
_Avoid_: Book content, copied prose

## Relationships

- A **Project-Backed Mastery Loop** contains **Theory for Mastery** before and during implementation.
- **Exercise-First Pressure** reinforces **Theory for Mastery** through frequent code.
- A project milestone is not complete until the learner has passed relevant exercises and satisfied the milestone rubric.
- The **First Slice Language** proves the model without making the initial build depend on every runtime and language server.
- A **Soft Lock** guides the learner through prerequisites while preserving freedom to jump ahead.
- **Evidence-Based Mastery** uses project and exercise evidence rather than treating a read lesson as proof of competence.
- The UI should be a **Serious IDE/Book Hybrid** with **Restrained Gamification** layered on top.
- A **Local Learner Profile** owns progress and preferences without introducing real authentication.
- A **Personal Study Note** is private learner state, not authored curriculum.
- A **Source Reference** points to external book coverage; it is not a copy of book text.

## Example dialogue

> **Dev:** "Should the learner read every lesson before touching a project?"
> **Domain expert:** "No. The **Project-Backed Mastery Loop** should pull in the theory and exercises needed for the next milestone, then make the learner prove the skill in code."

## Flagged ambiguities

- "Tutorialspoint-like" means clear, navigable, richly indexed tutorial structure; it does not mean passive reference-first learning is the primary path.
